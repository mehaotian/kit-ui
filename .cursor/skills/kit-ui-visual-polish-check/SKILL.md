# kit-ui 视觉精修检查技能

## 目标

- 对齐“精致、好看、可用”的组件库视觉标准。
- 降低不同组件间的视觉漂移。

## 检查维度

1. 色彩系统
   - 主色与功能色使用 `--k-*` 变量。
   - 不新增硬编码历史色值体系。
2. 间距与圆角
   - 使用主题间距变量，不出现随意魔法值堆叠。
   - 组件的默认圆角与主题保持一致。
3. 状态反馈
   - `normal/disabled/loading/active` 状态可区分。
   - 交互反馈不过度、不突兀，避免平台不兼容动画。
4. 文本可读性
   - 字号层级清晰。
   - 文本颜色对比度合理。
5. 演示页表达
   - 每个关键能力有可视化 demo。
   - 说明文案与实际行为一致。

## 检查流程

1. 读取目标组件与对应 demo 页。
2. 标记视觉不一致点与可能回归点。
3. 先修高优先级问题（色彩错误、状态缺失、可读性问题）。
4. 再修中优先级问题（间距、对齐、动效节奏）。
5. 形成“修改项 + 建议项”结论。

## 输出要求

- 按严重级别输出问题列表。
- 每项问题附带具体文件路径与修复建议。
