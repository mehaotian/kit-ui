# kit-ui 文档同步技能

## 目标

- 避免代码改了但 README、demo、清单未更新。
- 统一文档更新路径，降低组件库维护成本。

## 同步范围

- 组件文档：`uni_modules/kit-ui/components/k-[name]/README.md`
- 演示页面：`pages/[name]/[name].uvue`
- 组件清单：`docs/组件开发清单.md`

## 执行步骤

1. 识别代码变更点：
   - props 默认值与类型
   - emits 事件与参数
   - slot 结构
   - 主题变量与样式状态
2. 更新 README：
   - API 表与示例同步最新实现。
   - 写清跨端差异与已知限制。
3. 更新演示页：
   - 至少覆盖基础、状态、边界场景。
   - 新增能力要有可交互示例。
4. 更新组件清单：
   - 状态、优先级、完成度描述与实际代码一致。
5. 完成一致性检查：
   - README 中的示例属性在组件中确实存在。
   - 演示页使用的属性不存在拼写漂移。

## 输出要求

- 列出“代码改动 -> 文档改动”映射。
- 标注仍待补充的文档项（若有）。
