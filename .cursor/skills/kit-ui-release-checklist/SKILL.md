# kit-ui 发版检查技能

## 目标

- 在当前“以 HBuilderX 手工发布为主”的阶段，建立稳定可复用的发版清单。

## 发版前检查

1. 组件状态：
   - 本次变更组件在 `pages` 有对应演示页可访问。
   - 关键交互（禁用、加载、主题）可正常展示。
2. 文档状态：
   - 变更组件 README 已同步。
   - `docs/组件开发清单.md` 状态更新。
3. 规范状态：
   - 无 `gap/grid/@media/#ifdef H5` 新增。
   - 主题变量与命名符合 `k-` 体系。
4. 版本状态：
   - 记录本次变更摘要。
   - 若有破坏性变更，明确迁移说明。
5. 风险确认：
   - 列出“已验证端”和“未验证端”。

## 发布结果模板

- 发布范围：`k-xxx`、`k-yyy`
- 改动类型：功能 / 修复 / 样式
- 已验证平台：WEB / MP-WEIXIN / APP（按实际填写）
- 已知风险：无 / 列表

## 输出要求

- 必须提供可复制的发布检查结论。
- 若检查未通过，明确阻断项与建议处理顺序。
