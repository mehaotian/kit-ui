# kit-ui 规则冲突治理清单（一期）

## 文档目的

- 固化历史规则冲突，建立单一规范口径。
- 本文是一期升级期间的唯一冲突裁决依据。
- 后续 `.cursor/rules` 与 `.cursor/skills` 必须与本文保持一致。

## 冲突项与统一口径

| 冲突主题 | 历史冲突来源 | 统一口径（一期生效） |
| --- | --- | --- |
| 条件编译标识 | 同时出现 `H5` 与 `WEB` 写法 | 统一使用 `WEB`，不再新增 `H5` 写法 |
| 布局能力 | 部分文档示例使用 `grid` 和 `gap` | 组件库与 demo 页统一使用 `flex`，禁止 `grid` 与 `gap` |
| 响应式方案 | 有 `@media` 示例，也有“用 UTS 计算”约束 | 禁用 `@media`，通过 `uni.getWindowInfo` + 条件逻辑处理 |
| UTS 比较运算 | 示例同时出现 `===` 和 `==` | UTS 场景统一使用 `==` / `!=` |
| 对象类型定义 | 同时存在 `interface` 与“对象用 type”约束 | UTS 对象类型优先 `type`；`interface` 仅用于接口语义 |
| 字体样式作用域 | 有些示例把文字样式写在 `view` 上 | `color/font-size/font-weight` 仅用于 `text/button` |
| 主题变量命名 | `$primary-color` 与 `$k-color-primary` 混用 | 统一 `k-` 前缀变量体系（如 `$k-color-primary`、`--k-color-primary`） |
| 主题主色 | 旧文档仍有 `#007aff` | 统一主色为 `#6366f1` |
| 全局样式污染 | 部分示例允许全局覆盖 | 禁止新增全局污染性样式，组件样式局部化 |
| 规则职责边界 | rules 与 docs/模板内容大量重复 | rules 只保留硬约束，流程步骤迁移到 skills |

## 一期硬约束基线

1. 页面与组件文件后缀统一为 `.uvue`，UTS 逻辑写在 `<script setup lang="uts">`。
2. 页面新增时必须同步 `pages.json`。
3. 条件编译统一使用 `WEB / APP / MP-*`。
4. 组件与 demo 布局统一 `flex`，不使用 `grid`、`gap`、复杂伪类。
5. 字体相关样式只写在 `text` 或 `button`。
6. 不使用 `@media`，改为 UTS 逻辑控制。
7. UTS 代码用 `==` / `!=`，并显式处理空值与类型转换。
8. 主题变量统一 `k-` 命名，禁止新增旧命名体系。
9. 规则文件只描述“必须遵守”，不再堆叠长模板。
10. 开发流程统一走项目 skills，避免重复写法。

## 规则与技能分工

- `rules`：硬约束、禁用项、目录与命名红线。
- `skills`：组件创建、文档同步、跨端检查、发布检查、视觉验收、MCP 使用。

## 生效范围

- 生效目录：仓库根目录下所有前端代码与文档。
- 生效时间：本文件合入后立即生效。
