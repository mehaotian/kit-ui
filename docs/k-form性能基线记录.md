# k-form 性能基线记录

> 目标：建立 `k-form` 校验链路的可追踪性能基线，作为后续优化对比依据。  
> 口径：优先记录 WEB 数据，APP / 微信 MP 记录参考值。  
> 目标阈值（参考）：20 字段同步校验耗时 < 50ms（WEB）。

---

## 1. 测试场景定义

- 场景 A：20 字段 `validate()`（同步规则）。
- 场景 B：20 字段 `validateAsync()`（含少量异步规则，网络命中本地兜底）。
- 场景 C：`change -> blur` 连续触发（验证防抖与竞态下的响应稳定性）。

## 4. 回填规范

- WEB：建议使用浏览器 Performance 面板或时间戳埋点统计平均值（至少 5 次）。
- APP / 微信 MP：记录“平均耗时区间 + 是否有卡顿/抖动”。
- 若超过阈值，需补充：
  - 复现步骤
  - 规则配置
  - 优化建议（如规则拆分、触发策略调整、字段分段校验）。
