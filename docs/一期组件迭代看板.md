# kit-ui 一期组件迭代看板（可直接拆 Issue）

## 使用说明

- 本看板基于 `docs/组件现状盘点与优先级规划.md` 与 `docs/组件开发清单.md`。
- 迭代目标按「先可用、再完善」推进；**Slider / Rate / Upload / DatePicker / ActionSheet / Progress / Alert / Notification / Skeleton / Swiper / Picker** 已纳入一期高优，不再视为二期组件。
- 每个任务默认包含四类子任务：组件实现、demo 页面、README、跨端自测。

## 暂不纳入一期

### 业务耦合型

- `k-goods-card`、`k-goods-list`、`k-cart`、`k-submit-bar`、`k-address-picker` 等

### 低复用或沉底能力

- `k-link`、`k-copy`、`k-share`、`k-scroll-view`

### 高复杂预研类（二期）

- 图表、地图、富文本、`k-pull-refresh`、`k-virtual-list`

## Issue 模板（建议）

```base
标题：feat(组件): 新增 k-xxx 组件（一期）

验收标准：
- [ ] 组件 API 定义完成（props/emits/slots）
- [ ] demo 页面已创建并注册 pages.json
- [ ] README 已补齐并与实现一致
- [ ] 完成 APP / WEB / MP 冒烟验证
- [ ] 满足 kit-ui 一期 rules 红线要求
```
